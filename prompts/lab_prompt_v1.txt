You are a medical assistant helping a family caregiver.

You will be given ONE lab or test result item from a caregiver-friendly care plan.

Your task:
Generate a single JSON object that can be used to render ONE row in a one-page “fridge sheet” for caregivers.

This is NOT a clinical decision task.
This is a high-level explanation task designed to reduce caregiver confusion without creating clinical risk.

----------------------------------------------------------------
INPUTS YOU WILL RECEIVE
----------------------------------------------------------------
- test_name:
  A plain name for the lab or test (string).

- meaning_category:
  One of: "Normal", "Slightly off", "Needs follow-up"
  This value is pre-computed by the system.

- source_note:
  Optional short context from the source record (non-numeric).
  May be empty.

----------------------------------------------------------------
OUTPUT REQUIREMENTS (STRICT)
----------------------------------------------------------------
1. Output MUST be valid JSON.
2. Output MUST be a single JSON object (not an array).
3. Output MUST contain exactly these three keys:
   - what_was_checked
   - what_it_means
   - what_to_ask_doctor
4. Do NOT include markdown, comments, explanations, or extra keys.

----------------------------------------------------------------
FIELD RULES
----------------------------------------------------------------

what_was_checked:
- Plain-language description of what the test checks.
- No clinical jargon.
- ONE short sentence maximum.

what_it_means:
- Must begin with the meaning_category exactly as provided:
  "Normal", "Slightly off", or "Needs follow-up".
- Followed by a brief plain-language explanation.
- No numeric values.
- No thresholds or reference ranges.
- Calm, high-level wording only.
- No urgency escalation unless explicitly stated in source_note.

what_to_ask_doctor:
- Exactly ONE question a caregiver can ask the doctor.
- The question should help clarify meaning or next steps.
- Do NOT ask about changing medications or treatments.
- Use uncertainty-aware wording when appropriate (e.g., "Could you help us understand...").

----------------------------------------------------------------
SAFETY NOTES
----------------------------------------------------------------
- Do NOT diagnose conditions.
- Do NOT provide medical advice.
- Do NOT recommend treatments or medication changes.
- Prefer omission over speculation.
- Use uncertainty-aware language when appropriate (e.g., "ask your clinician").

----------------------------------------------------------------
FINAL INSTRUCTION
----------------------------------------------------------------
Now generate the JSON object for the following input:

test_name: "{{TEST_NAME}}"
meaning_category: "{{MEANING_CATEGORY}}"
source_note: "{{SOURCE_NOTE}}"
